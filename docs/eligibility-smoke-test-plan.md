# 자격검토 E2E Smoke Test 설계

## 목표

- 공개 운영 URL 기준으로 `자격검토 → 판정 리포트 → 상담 신청` 흐름이 끊기지 않는지 빠르게 검증한다.
- 프런트 fallback만 살아 있고 운영 API가 깨진 상태를 조기에 잡는다.
- 기능 전체를 깊게 보증하기보다, 배포 직후 반복 확인 가능한 핵심 동선의 이상 유무를 체크한다.

## 범위

- 운영 배포 URL: `https://reporeview.vercel.app`
- 포함
  - `/api/health`
  - `/api/programs`
  - `/check` 공통 질문
  - `/check` 후속 질문
  - 판정 리포트 렌더링
  - 상담 폼 제출 성공
- 제외
  - 숨김 상태인 `/apply`
  - 관리자용 데이터 수정 흐름
  - 모든 제도 조합에 대한 완전한 규칙 검증

## 스모크 시나리오

### 시나리오 1

- 이름: 운영 API 기본 응답 확인
- 목적: 운영 API가 `FUNCTION_INVOCATION_FAILED` 없이 살아 있는지 점검
- 확인 항목
  - `/api/health` 200
  - `ok === true`
  - `programCount > 0`
  - `/api/programs` 200
  - `programs.length > 0`

### 시나리오 2

- 이름: 지역고용촉진지원금 보완 필요 경로
- 목적: 공통 질문, 후속 질문, 판정 리포트, 상담 성공까지의 대표 happy path 검증
- 입력
  - 기업 규모: 우선지원대상기업
  - 상황: 고용위기지역 이전·신설·증설
  - 상시 근로자 수: 30인 이상 100인 미만
  - 소재지: 비수도권
  - 계획 신고: 아직 미신고, 준비 가능
  - 사업 유형: 이전·신설·증설 해당
  - 지역 거주자: 예
  - 6개월 유지: 예
- 기대값
  - 결과 제도명: 지역고용촉진지원금
  - 상태: 보완 필요
  - 보완 항목: 지역고용계획 신고 선행
  - 상담 폼 성공
  - `POST /api/consultation-leads` 201

### 시나리오 3

- 이름: 정규직 전환 지원금 제외 경로
- 목적: 공통 조건으로 제외되는 판정이 화면에 정확히 드러나는지 검증
- 입력
  - 기업 규모: 우선지원대상기업
  - 상황: 비정규직 정규직 전환
  - 상시 근로자 수: 100인 이상
  - 소재지: 수도권
  - 나머지 후속 질문: 모두 예
- 기대값
  - 결과 제도명: 정규직 전환 지원금
  - 상태: 현재 제외
  - 보완 항목: 5인 이상 30인 미만 기업 요건

## 운영 원칙

- 이메일은 실제 발송 대신 Playwright 네트워크 route로 `EmailJS` 응답을 모의한다.
- 상담 리드 저장 API는 실제 운영 `/api/consultation-leads` 호출을 통과시켜 배포 연결 상태를 확인한다.
- 선택지 클릭은 `data-testid` 기반으로 고정해 UI 문구 일부가 바뀌어도 테스트가 쉽게 깨지지 않게 한다.

## 실행 방법

```powershell
npx.cmd playwright test
```

특정 운영 URL을 바꾸고 싶으면 아래처럼 실행한다.

```powershell
$env:PLAYWRIGHT_BASE_URL="https://reporeview.vercel.app"
npx.cmd playwright test
```

## 현재 한계

- 브라우저 레벨 스모크 테스트이므로, 모든 지원금 규칙 조합을 보장하지는 않는다.
- EmailJS는 모의 응답을 사용하므로 실제 외부 메일 발송 성공 자체는 별도 수동 점검이 필요하다.
- 상담 리드 저장소가 영속 DB가 아닌 경우, 저장 성공은 API 동작 확인 수준으로 해석해야 한다.
