# 자격검토 E2E Smoke Test 설계

## 목표

- 공개 운영 URL 기준으로 `자격검토 → 판정 리포트 → 상담 신청` 흐름이 끊기지 않는지 빠르게 검증한다.
- 프런트 fallback만 살아 있고 운영 API가 깨진 상태를 조기에 잡는다.
- 기능 전체를 깊게 보증하기보다, 배포 직후 반복 확인 가능한 핵심 동선의 이상 유무를 체크한다.

## 범위

- 운영 배포 URL: `https://reporeview.vercel.app`
- 포함
  - `/api/health`
  - `/api/programs`
  - `/check` 공통 질문
  - `/check` 후속 질문
  - 판정 리포트 렌더링
  - 상담 폼 제출 성공
  - DB 게시 규칙 기반 워라밸/출산육아/고용유지 묶음 판정
- 제외
  - 숨김 상태인 `/apply`
  - 관리자용 데이터 수정 흐름
  - 모든 제도 조합에 대한 완전한 규칙 검증

## 운영 원칙

- 이메일은 실제 발송 대신 Playwright 네트워크 route로 `EmailJS` 응답을 모의한다.
- 상담 리드 저장 API는 실제 운영 `/api/consultation-leads` 호출을 통과시켜 배포 연결 상태를 확인한다.
- 선택지 클릭은 `data-testid` 기반으로 고정해 UI 문구 일부가 바뀌어도 테스트가 쉽게 깨지지 않게 한다.

## 실행 방법

```powershell
npx.cmd playwright test
```

특정 운영 URL을 바꾸고 싶으면 아래처럼 실행한다.

```powershell
$env:PLAYWRIGHT_BASE_URL="https://reporeview.vercel.app"
npx.cmd playwright test
```

## 자동 실행

- GitHub Actions workflow: `.github/workflows/deployment-smoke.yml`
- 트리거
  - Vercel 등에서 전달되는 `deployment_status: success`
  - 수동 실행 `workflow_dispatch`
- 동작
  - 배포 URL의 `/api/health` 준비 상태 대기
  - Playwright Chromium 설치
- 운영 URL 기준 smoke suite 실행
- 실패 시 Playwright 결과 아티팩트 업로드

## 관리자 운영 스모크

- 파일: `e2e/admin-lifecycle.spec.ts`
- 기본 동작
  - 관리자 로그인
  - 문서 draft 생성
  - 문서 상세 재조회
  - draft override 저장
- 선택 동작
  - `ALLOW_ADMIN_PUBLISH_SMOKE=true`일 때만 publish까지 수행
- 필요한 환경변수
  - `ADMIN_TEST_EMAIL`
  - `ADMIN_TEST_PASSWORD`
  - `ALLOW_ADMIN_PUBLISH_SMOKE`

- 실행 예시

```powershell
$env:ADMIN_TEST_EMAIL="admin-codex@winners.local"
$env:ADMIN_TEST_PASSWORD="Winners!2026Admin#"
$env:ALLOW_ADMIN_PUBLISH_SMOKE="true"
npx.cmd playwright test e2e/admin-lifecycle.spec.ts --reporter=list
```

## 현재 한계

- 브라우저 레벨 스모크 테스트이므로, 모든 지원금 규칙 조합을 보장하지는 않는다.
- EmailJS는 모의 응답을 사용하므로 실제 외부 메일 발송 성공 자체는 별도 수동 점검이 필요하다.
- 상담 리드 저장소가 영속 DB가 아닌 경우, 저장 성공은 API 동작 확인 수준으로 해석해야 한다.
- 영속 저장과 관리자 운영화가 필요해지는 시점부터는 외부 DB 도입을 검토해야 하며, 현재 smoke 단계에서는 필수 아님.
