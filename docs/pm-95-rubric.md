# 95점 PM 루브릭

## 목적

- 이 문서는 `고용장려금 탐색 -> 자격 자가진단 -> 후속 상담 전환` 흐름을 기준으로 제품 완성도를 평가하기 위한 운영용 루브릭이다.
- 점수는 단순 인상평가가 아니라 `신뢰`, `전환`, `성능`, `운영 일관성`을 중심으로 채점한다.
- 95점은 "예쁘다"가 아니라 "상용 제품으로 추천 가능하다"는 수준을 뜻한다.

## 총점 구조

| 항목 | 배점 | 핵심 질문 |
|------|------|-----------|
| 문제 해결 적합성 | 20 | 사용자가 이 사이트의 가치와 목적을 즉시 이해하는가 |
| 자격검토 결과 신뢰 UX | 25 | 결과를 억울함이나 혼란 없이 이해하고 다음 행동을 알 수 있는가 |
| 전환 설계와 상담 UX | 15 | 상담이 영업 압박이 아니라 합리적 다음 단계로 느껴지는가 |
| 정보 구조와 카피 완성도 | 15 | 메뉴, 카드, CTA가 하나의 제품 톤으로 통일되는가 |
| 성능과 반응성 | 10 | 첫 진입과 단계 전환에서 멈춤이 거의 없는가 |
| 운영 완성도와 배포 일관성 | 10 | Preview/Production과 환경변수가 예측 가능하게 동작하는가 |
| 신뢰/컴플라이언스 표현 | 5 | 면책과 신뢰를 동시에 해치지 않는가 |

## 1. 문제 해결 적합성 — 20점

### 채점 기준

- 0~5점: 서비스 목적이 흐릿하고 방문 직후 무엇을 해야 할지 알기 어렵다.
- 6~10점: 홈과 자격검토가 연결되지만 진입 이유나 기대값이 약하다.
- 11~15점: 홈, 목록, 상세, 자격검토가 하나의 흐름으로 연결된다.
- 16~20점: 홈 첫 화면만 보고도 `누구를 위한 서비스인지`, `무엇을 해결하는지`, `왜 지금 봐야 하는지`가 분명하다.

### 95점 기준

- 사용자가 10초 안에 `내가 볼 가치가 있는지`, `무엇을 해야 하는지`, `얼마나 신뢰할 수 있는지`를 이해해야 한다.
- 홈의 CTA와 `/check` 진입 의도가 논리적으로 연결되어야 한다.

## 2. 자격검토 결과 신뢰 UX — 25점

### 채점 기준

- 0~7점: 결과 상태가 모호하거나 서로 겹친다.
- 8~14점: 상태는 보이지만 이유와 다음 행동이 충분히 분리되지 않는다.
- 15~20점: 상태, 이유, 다음 행동이 대부분 이해 가능하다.
- 21~25점: 어떤 결과를 받아도 사용자가 비난받는 느낌 없이 `이유 -> 현재 상태 -> 다음 행동`을 바로 이해한다.

### 95점 기준

- 상태 라벨 `신청 가능`, `조금 더 확인 필요`, `조건 다시 확인`, `추가 확인 필요`가 서로 겹치지 않아야 한다.
- 결과 카드마다 `왜 이런 결과인지`, `무엇을 먼저 보면 되는지`, `자동 판정의 한계가 어디까지인지`가 분명해야 한다.
- 결과는 "심사"보다 "안내"처럼 느껴져야 한다.

## 3. 전환 설계와 상담 UX — 15점

### 채점 기준

- 0~4점: 상담 CTA가 압박형이거나 결과 반복 문장이 많다.
- 5~8점: 상담으로 이어지지만 결과와 CTA의 위계가 어색하다.
- 9~12점: 결과 요약과 보완 항목이 정리되고 CTA가 비교적 자연스럽다.
- 13~15점: 상담이 판매가 아니라 합리적 다음 단계로 인식된다.

### 95점 기준

- 상담 박스는 결과를 로그처럼 길게 반복 설명하지 않는다.
- 결과 요약과 보완 항목은 시각적으로 분리된다.
- 상담 CTA는 `지금 상담받아야 한다`가 아니라 `필요하면 이어서 정리받을 수 있다`로 읽혀야 한다.

## 4. 정보 구조와 카피 완성도 — 15점

### 채점 기준

- 0~4점: 메뉴, 카드, CTA 문구가 각각 다른 제품처럼 보인다.
- 5~8점: 전체 톤은 맞지만 일부가 보고서 문체나 내부 시스템 문구처럼 읽힌다.
- 9~12점: 대부분의 문구가 안내형 톤으로 정리된다.
- 13~15점: 빅테크 PM이 봐도 `내부 시스템 문구`, `보고서 문체`, `영업 문구`가 거의 남아 있지 않다.

### 95점 기준

- 전문성은 유지하되, 실무자가 바로 이해할 생활어를 사용한다.
- 같은 의미를 다른 말로 반복하지 않는다.
- 홈, 상세, 자격검토, 상담이 하나의 제품 톤으로 느껴져야 한다.

## 5. 성능과 반응성 — 10점

### 채점 기준

- 0~2점: 첫 진입과 단계 전환에서 뚜렷한 멈춤이 반복된다.
- 3~5점: 느린 구간이 있지만 흐름을 방해하지는 않는다.
- 6~8점: 첫 진입과 전환이 비교적 빠르고 fallback이 자연스럽다.
- 9~10점: 사용자가 흐름 중 `멈췄다`고 느끼는 구간이 거의 없다.

### 95점 기준

- `/check` 첫 진입 시 첫 질문이 즉시 보여야 한다.
- 단계 전환 시 300~500ms 이상 멈춘 느낌이 최소화되어야 한다.
- 결과 화면 진입은 낙관적 렌더 또는 즉시성 있는 피드백을 제공해야 한다.

## 6. 운영 완성도와 배포 일관성 — 10점

### 채점 기준

- 0~2점: Preview/Production이 자주 엇갈리고 환경 문제를 UX 문제로 오해하게 만든다.
- 3~5점: 운영은 가능하지만 배포별 동작 차이가 자주 설명 필요하다.
- 6~8점: 배포 및 환경 차이가 관리 가능하고 예측 가능하다.
- 9~10점: 최신 배포, 환경변수, API fallback, 상담 전송이 일관되게 동작한다.

### 95점 기준

- Preview와 Production의 동작 차이가 설명 가능해야 한다.
- 환경변수, 이메일 전송, API fallback, DB 규칙 동작이 배포 환경별로 예측 가능해야 한다.
- 배포 후 `보이는 화면`, `실제 이메일 동작`, `판정 결과`가 서로 어긋나지 않아야 한다.

## 7. 신뢰/컴플라이언스 표현 — 5점

### 채점 기준

- 0~1점: 면책 표현이 변명처럼 보이거나 과도하게 위축된다.
- 2~3점: 참고용 문구는 있으나 신뢰 표현과 잘 조화되지 않는다.
- 4~5점: 참고용 판정임을 전달하면서도 서비스 신뢰를 해치지 않는다.

### 95점 기준

- 실제 신청 전 확인 필요성을 과하지 않게 명시한다.
- 법적 리스크를 피하면서도 사용자가 `이 서비스는 책임감 있게 안내한다`고 느껴야 한다.

## 95점 게이트

아래 중 하나라도 미달이면 총점이 95를 넘지 못한다.

- `자격검토 결과 신뢰 UX` 22점 미만 불가
- `전환 설계와 상담 UX` 13점 미만 불가
- `성능과 반응성` 8점 미만 불가
- `운영 완성도와 배포 일관성` 8점 미만 불가
- 치명적 결함 0건

### 치명적 결함 예시

- 최신 배포에서 예전 문구와 최신 문구가 섞여 보인다.
- 상담 성공처럼 보이지만 실제 이메일이 오지 않는다.
- 상태 체계가 사용자 입장에서 모순된다.
- 첫 진입 또는 결과 전환에서 뚜렷한 멈춤이 반복된다.

## 평가 프로토콜

### 1차 정성 리뷰

- 빅테크 PM 관점으로 10분 사용성 리뷰 수행
- 홈 첫 화면, `/subsidies`, `/check`, 상담 제출 전까지 이어서 확인

### 2차 시나리오 리뷰

- `신청 가능` 시나리오 1회
- `조금 더 확인 필요` 시나리오 1회
- `조건 다시 확인` 시나리오 1회
- `추가 확인 필요` 시나리오 1회

### 3차 운영 리뷰

- Preview와 Production에서 동일 시나리오 비교
- 문구, 상태, 상담 박스 구조, 이메일 동작 차이 확인

### 4차 전환 리뷰

- 상담 제출 1회
- 이메일 수신 1회
- 결과 요약 표현 검토 1회

## 운영용 체크리스트

### 제품 메시지

- 홈 첫 화면에서 서비스 목적이 10초 안에 이해된다.
- CTA가 자격검토 흐름과 논리적으로 연결된다.
- 홈, 목록, 상세, 자격검토의 톤이 하나로 느껴진다.

### 결과 신뢰

- 결과 상태 4종이 서로 겹치지 않는다.
- 결과 카드에서 이유, 현재 상태, 다음 행동이 분리되어 보인다.
- 사용자가 비난받거나 탈락 통보받는 느낌이 들지 않는다.

### 상담 전환

- 상담 박스가 결과를 한 줄 로그처럼 길게 반복하지 않는다.
- 현재 확인 결과와 보완 항목이 분리되어 보인다.
- CTA가 자연스럽고 압박감이 없다.

### 성능

- `/check` 첫 질문이 즉시 보인다.
- 단계 전환이 끊기지 않는다.
- 결과 화면 진입이 자연스럽다.

### 운영

- 최신 Preview가 최신 커밋을 보고 있다.
- Production과 Preview의 차이가 설명 가능하다.
- EmailJS 환경변수가 Preview/Production에 정확히 들어가 있다.
- 상담 제출 성공 시 실제 이메일도 도착한다.

## 95점 판정 조건

- 총점 95 이상
- 게이트 전부 통과
- 치명적 결함 0건
- 사용자가 `무엇을 해야 하는지`, `왜 이런 결과인지`, `상담이 왜 필요한지`를 추가 설명 없이 이해한다.
- PM 관점에서 "잘 만든 도메인 툴"이 아니라 "출시 가능한 제품"으로 평가할 수 있다.

## 기본 가정

- 이 루브릭은 `노무법인 위너스의 고용장려금 안내/진단/상담 전환 사이트`에 맞춘다.
- 시각 디자인 미감보다 `신뢰`, `전환`, `성능`, `운영 일관성`을 더 중요하게 본다.
- 95점은 "매우 좋다"보다 더 높은 기준이며, 실제 출시 품질 판단에 사용하는 수준으로 본다.
